# J · 타겟 공고 노출 — 구현 설계 (2026-07-18)

> 설문 확정: 데이터=다대다 / 배정=**규칙(자동)+수동(일괄) 둘 다** / 실무자 편의 최우선.
> 전제: 공고 5~10개 동시, 각 타깃 풀이 교집합·여집합. 기본 동작('전체 노출')은 불변.

## 1. 스키마 (마이그레이션 신규)
- `jobs.exposure text default 'all'` — `'all'`(전체 노출, 기존 동작) | `'targeted'`(지정 노출). check 제약.
- `jobs.exposure_rule jsonb null` — 자동 노출 **조건**. 파이프라인 필터 스키마 재사용: `{ sido?, sigungu?, vehicle?('확정'|'도보'|'미확인'), availability?, suntopDone?, cohortMonths?, ... }`. null이면 규칙 없음(수동만).
- `job_exposure_targets(id, job_id, applicant_id, mode text 'include'|'exclude', added_by, created_at, unique(job_id, applicant_id))` — 수동 오버라이드.
- RLS on(서버 전용). FK/인덱스 job_id·applicant_id.

## 2. 유효 노출 판정 (지정 노출 공고 J, 지원자 A)
```
if (J,A,'exclude') 존재            → 노출 안 함   (명시적 제외가 최우선)
else if (J,A,'include') 존재       → 노출
else if A가 J.exposure_rule 매칭   → 노출
else                               → 노출 안 함
```
- 전체 노출(all) 공고는 항상 노출(현행 유지). 시스템·마감 공고는 기존대로 제외.
- 규칙 매칭 = applicant 필드 서버 평가. **파이프라인 필터와 동일 의미 유지 필수**(드리프트 주의) → 공용 매처 lib로 단일화.

## 3. UX 3면 (실무자 편의)
1. **공고 폼(Jobs.tsx)**: 노출 범위 토글 + 지정 시 **규칙 빌더**(파이프라인 필터 컴포넌트 재사용) + 실시간 "규칙 해당 N명" 미리보기. **확정 뉘앙스 금지**('노출 대상'이지 배정 아님).
2. **파이프라인 일괄(Pipeline.tsx)**: 필터·세그먼트로 선택 → **"노출: 공고에 추가/제외"**(다중 공고 선택) → include/exclude 행 기록. 기존 `addSelectedToJob` 패턴 확장. 교집합을 한 번에 처리하는 핵심 동선.
3. **공고 상세/후보 패널**: 유효 노출 명단(규칙+수동−제외) + 인원수 + 개별 제거(→exclude 기록). 규칙 대상인지/수동인지 배지 구분.

## 4. pull · interest
- `pull GET(/api/pool/[token])`: 지정 노출 공고는 유효 노출 계산해 필터(노출 대상만 그 공고 노출). exposure/규칙은 응답 미노출.
- `interest/notify`: 지정 노출인데 미대상이면 기존 불투명 400("모집 마감") 유지(공고 존재 노출 방지). 관심 클릭 시 노출→`job_candidates` 정식 후보 전환.
- **interested 산식 보정**(기존 J 계획 유지): jc 참여 마커/`interest_click` 기준으로 '접수' 판정(피커/노출 대상이 전부 ✓로 잠기지 않게).

## 5. 빌드 순서
1. 마이그레이션(스키마) → prod 적용
2. `lib/exposure.ts` — 규칙 매처(applicant↔rule) + 유효 노출 판정(단건/배치). 파이프라인 필터 의미와 단일 소스화
3. pull GET·interest 게이팅
4. 어드민 API — 규칙 저장(jobs PATCH), 일괄 include/exclude, 공고별 유효 명단 조회
5. UI 3면(공고 폼 규칙빌더 · 파이프라인 일괄 · 공고 상세 명단)
6. 검증(실데이터 노출 계산) + 적대 리뷰(노출 누수/오노출·규칙-필터 드리프트)

## 6. 리스크
- 규칙-필터 **의미 드리프트** → 공용 매처 lib 단일화로 방지.
- **exclude 우선** 불변식(명시 제외가 include·규칙 다 이김).
- 규칙 평가 성능: pull은 단건(싸다), 미리보기·명단은 배치(인재풀 규모 수백 → OK, 수천 시 인덱스/캐시 검토).
- 시스템/마감 공고 가드 유지.
