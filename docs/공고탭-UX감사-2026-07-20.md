# 채용공고 관리 탭 UX 감사 (2026-07-20)

> 4렌즈(실무 동선·모달 복잡도·일관성·상태 표시) 감사 → 사실검증 통과 31건(중복 정리 후 19개 주제).
> 전 항목 파일:라인 검증 완료. 심각도 = 실무 피해 기준.

## A. 실사용 차단 버그 — 기능이 실제로 안 됨 ★최우선
1. **internal 근무시간 = 등록 불가 지뢰** — UI는 자유 텍스트(placeholder "월~토 오전 7시~")인데 서버 POST/PATCH가 slot 4-enum으로 검증해 400. 에러 문구도 'slot'이라 어느 칸인지 못 찾음. (Jobs.tsx:1650·1939 ↔ jobs API slot 검증)
2. **마감시각 지난 공고 '재개' 무효** — 재개 PATCH가 status만 바꾸고 closes_at은 그대로 → 여전히 '마감됨'. 반대로 실질 마감 공고에 액션이 '공고 마감'뿐(이미 마감인데). 정기 라인 재모집 동선 파탄. (Jobs.tsx:999-1049·1514 ↔ [id]/route.ts:146)
3. **수정 한 번에 화주사 귀속 소실** — 수정 모달엔 화주사 셀렉트가 없는데 저장 시 branch_id를 무조건 전송 → 지점 없는 공고는 서버가 client_id까지 null로 소거 → 화주사 필터에서 공고 증발. (Jobs.tsx:963 ↔ [id]/route.ts:166)

## B. 작업물 파기 — 확인 없는 닫힘
4. **등록 모달 X/'닫기' = 즉시 전체 파기** — 10분 다듬은 3채널 초안·근무상세·노출규칙이 경고 없이 소멸(AI 재생성 비용도 재발). 수정 모달은 **backdrop 클릭만으로** 편집분 폐기. '파괴적 액션 확인 모달' 원칙이 자기 데이터엔 미적용. (Jobs.tsx:1557·1837·1858)

## C. 발송 사고 위험
5. **external 공고에도 '대기자에게 안내' 노출** — 발송 문자의 맞춤링크에 external 공고는 안 뜸 → "공고가 어디 있냐" 회신 폭탄. 게시 링크 버튼은 internal 게이팅돼 있는데 확성기만 누락. (Jobs.tsx:1493, announce-targets recruit_mode 미검사)
6. **미발송 후보 일괄 문자(dispatchUnsent)가 원클릭** — 확인·미리보기 없이 실SMS 대량 발송. 같은 화면의 마감/새공고 안내는 모두 모달을 거침. (Jobs.tsx:2151)

## D. J(타겟 노출) 신규 기능의 UX 부채
7. **draft vs 즉시반영 혼재** — 규칙은 저장 눌러야 반영인데 명단 '제외/복원'은 즉시 DB 커밋. 취소해도 제외가 남아 나중에 "왜 저 분한테 공고가 안 보이죠" 미스터리. (ExposureEditor:165 ↔ Jobs:1911)
8. **'저장 후 새로고침' 안내가 실행 불가** — 저장 버튼이 모달을 닫아버림. + 위(draft 기준 미리보기 N명)와 아래(저장 기준 명단 M명) 숫자 2개가 기준이 달라 나란히 모순. (ExposureEditor:310·315 ↔ Jobs:990)
9. **등록 모달에서 노출 설정 찾기 불가** — '근무 상세 · AI 응대 근거' 접이식 맨 아래 숨음(라벨에 '노출' 없음). 수정 모달은 최상위라 위치도 불일치. 복제 시 접이식 자동 오픈 조건에 exposure 누락 → targeted 복제가 안 보인 채 등록됨. (Jobs:1713·1639·781)
10. **external 전환 시 targeted 고아** — 에디터만 숨고 exposureDraft='targeted'는 그대로 저장 → '공개 모집'+'지정 노출' 배지 동시, 끌 UI 없음. (Jobs:1910·971·817)
11. **'지정 노출 + 대상 0명' 위험 상태 상시 표시 없음** — 카드 배지에 인원수 없음(데이터는 이미 내려옴). 신규 등록에서 빈 규칙 경고 없음(수동 지정은 등록 후에만 가능하다는 안내도 수정 모달 전용). (Jobs:1371, ExposureEditor:289)
12. **미리보기 에러가 '규칙 비어 있음'으로 위장** — POST 실패 시 setPreview(null) → 규칙이 차 있는데 "규칙이 비어 있어요" 표시. (ExposureEditor:141·288)
13. '지정 노출' 배지 색 = '공개 모집' 배지 색(완전 동일 파랑) + 'AI 응대 중' pill도 같은 계열. (Jobs:1372·241·1438)
14. 모집 방식 desc가 이미 '노출' 어휘 사용("맞춤링크에 노출") → 바로 아래 '노출 범위'와 개념 충돌. (Jobs:240 ↔ ExposureEditor:202)

## E. 모달 구조 — 정보 과부하
15. **등록 모달 제목 입력 없음** — 제목이 AI 응답/본문 첫 줄에서 몰래 결정, pull 카드·안내 문자에까지 나감 → 매 등록마다 수정 왕복. (Jobs:795, 등록 모달에 postingTitle 입력 부재)
16. **등록 모달 푸터 = 라벨 없는 컨트롤 8~9개 벽** — 정원이 라벨 없는 72px 스피너(놓치면 1명 확정=충원 100%로 '마감' CTA 오작동). 화주사·지점도 푸터라 누락 빈발. (Jobs:1723-1828)
17. **수정 모달 ~24컨트롤 평면 나열(≈2,100px)** — 가장 흔한 '마감시각 연장'이 노출 규칙 빌더+명단을 다 지나야 나옴. 섹션 접이식 없음. (Jobs:1867-1994)
18. 급여 입력처 3곳 분산(푸터 단가형태/금액 · 접이식 textarea · 본문) + 등록/수정 위치 불일치. (Jobs:1778·1701·1968)

## F. 상태 표시
19. **마감 공고 충원율 0/N 왜곡** — 서버가 마감 공고를 confirmed 집계에서 제외 → 3/3 채워 마감한 공고가 '0/3'으로 표시. (route.ts:74 ↔ Jobs:1375)
20. **로딩/에러가 '공고 0건'으로 위장** — error/isLoading 미사용, 느린 아침에 "전체 공고 0건 + 이 상태의 공고가 없어요". (Jobs:639·1358·1525)
