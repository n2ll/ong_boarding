# 플로우·설정 UX 감사 (2026-07-20)

> 탭 간 여정·설정 화면 감사(32에이전트) → 사실검증 통과 24건(high 12/med 12). 공고탭 자체 감사(별도 문서)와 중복 제외.
> 사용자 문제의식("화주사·지점 설정이 기능을 못 좇아감 / 게재·수정·확정 플로우 불편")과 정확히 일치.

## 주제 A — 설정 화면 도달 불가 (최다 클러스터, 사용자 핵심 지적) ★
공고·확정이 **요구하는** 데이터를 **만들 UI 경로가 없다.**
- **A1 · /clients(화주사 관리) 죽은 화면**: 사이드바 숨김 + 코드 전역에 진입 링크 0(grep). URL 직접 입력만 도달. 그런데 공고 등록 폼·지점 폼·확정 지점 매칭이 전부 이 `clients` 테이블 의존. → **새 화주사 만들 길이 없어 화주사 미귀속 공고 양산**(필터·그룹핑서 영구 유실).
- **A2 · /branches(지점 관리) 숨은 진입점**: 유일 도달 경로가 '에이전트 두뇌' 탭의 AI 커버리지 카드 클릭 하나. '지점 관리' 멘탈모델로는 못 찾음. 지점은 공고 귀속·전역 필터·충원율 축인데.
- **A3 · 두 '화주사'의 혼동**: 사이드바에 보이는 '화주사·라인 현황'(/shippers)은 **옹매니징 읽기전용 미러**, 공고가 실제 참조하는 건 **별도 로컬 clients**(/clients, 숨김). 둘 사이 브릿지·설명 없음 → "방금 봤는데 왜 공고엔 없지?"
- **A4 · /team(현장매니저)·/slots(슬롯) 도달 불가**: external 만남장소 발송은 site_manager 필요한데 공고 폼에 현장매니저 셀렉터 없고 /team 숨김. 확정 모달 안내문이 참조하는 /slots도 숨김(실행 불가 안내).
- **A5 · 확정 지점이 자유 텍스트**: 확정 모달 '확정 지점'이 placeholder='예: 강남' 손입력인데, 지점 충원율 집계는 **등록 지점명 정확 일치** 매칭 → 오타 한 글자로 집계서 누락. 드롭다운 필요.

## 주제 B — 확정 경로 3개가 결과가 다름 (데이터 정합·사용자 핵심 지적) ★
- **B1 · 드래그/일괄 확정 = 반쪽 확정(위험)**: 칸반 '확정 인력' 열 드래그·리스트 일괄 상태변경은 `status`만 PATCH — 대상 공고 결속(current_job_id)·시작일·지점·슬롯 없음, 잔여 후보 자동정리·**그 공고 AI 정지 안 됨**. 상세 패널 확정 모달만 온전. → 가장 쉬운/눈에 띄는 경로가 가장 불완전. 확정했는데 충원율 0, **AI가 계속 지원 안내 발송**('붙었는데 왜 또 연락' 항의).
- **B2 · 일괄 '확정 인력'이 확인 모달 없음**: 재컨택 50명 선택 중 오클릭 → 확인창 없이 50명 즉시 확정, 대량 통계 오염, 일괄 취소 버튼 없음.
- **B3 · 확정 취소 동선 없음**: 상세에 '확정 취소' 없음. 잘못된 공고로 확정해도 정정 시 이전 공고 후보 paused 잔재·hired_at·시작일 잔존 → 리드타임 통계 왜곡.

## 주제 C — 확정 후속 안내 분절
- **C1**: 만남장소·첫날규칙 발송 UI가 '실시간 응대 > 확정 대기' 서브탭에만. 확정을 커밋하는 상세 모달은 '앱 안내' 체크박스만 → 신입은 만남장소·첫날규칙 발송 통째 누락하고 현장 보냄. 확정 대기 큐 건수도 사이드바·대시보드 어디에도 안 뜸(발견성 0).

## 주제 D — 게재 흐름 격차
- **D1**: AI 초안은 항상 3채널(당근·알바몬·SMS) 생성하는데 저장은 albamon 1개만 → 채널 특화 소실, 복제 시 3채널 동일 본문.
- **D2**: 초안 생성 API가 free-text만 받음 — 마스터의 화주사명·지점 ai_facts(집결지·시급) 미활용. 화주사/지점 선택이 초안 생성 뒤(푸터)라 순서도 어긋남.
- **D3**: 기본 모집방식 internal인데 초안 첫 탭은 당근(외부 채널) — 게시 링크는 external/both에만 떠 초안과 배포 채널 불일치.
- **D4**: 게시 링크 소스 목록에 당근 없음 → 당근 유입 소스 귀속 링크 못 만듦.

## 권장 우선순위
1. **주제 B(확정 정합)** — 데이터 오염·오발송 위험이라 최우선. 확정을 단일 경로(모달)로 수렴 + 확정 취소.
2. **주제 A(설정 도달)** — 사용자 핵심 지적. 숨김 나비 전면 복원보다 **필요 지점에 진입점 인라인**(공고 폼 옆 '+화주사/지점', 설정 페이지 링크) + /shippers↔/clients 명칭·관계 정리 + 확정 지점 드롭다운.
3. **주제 C(후속 안내 연결)** — 확정→후속 안내 한 흐름.
4. **주제 D(게재 다듬기)** — 채널 저장·초안 데이터 연계.
